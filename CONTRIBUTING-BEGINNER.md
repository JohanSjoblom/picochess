# Contributing to Picochess: a beginner-friendly path

You can help Picochess without being an experienced programmer. The most valuable start is a clearly described problem or improvement. A maintainer can help with the technical decisions.

## The simplest path: GitHub in your browser

1. [Create a bug report or improvement idea](https://github.com/JohanSjoblom/picochess/issues/new/choose). Write what you see, what you expected, and how someone can try it.
2. Wait for a maintainer to discuss the idea and, where suitable, add the `good first issue` label. This means the task has a bounded, beginner-appropriate starting point.
3. Click **Fork** on the repository page. GitHub creates your personal copy; this is where you can safely make changes.
4. In your fork, click **Code** then **Create codespace on _branch_**. Codespaces opens a complete cloud workspace in your browser—no local Python, editor, or Picochess installation is needed.
5. Read [CONTRIBUTING-AI.md](CONTRIBUTING-AI.md), describe the linked issue to your AI coding assistant, then review and test its work.
6. Use the workspace terminal to run `./scripts/submit-contribution`. It guides you through testing, committing, pushing, and creating a **draft pull request** to `JohanSjoblom/picochess`.

The pull request is a request for review, not a claim that the work is perfect. Draft PRs are encouraged. A maintainer will help review the technical details.

## What makes a good first issue?

Good first issues are small enough to describe and check. For example: a typo or confusing message, a missing documentation step, a small web-layout improvement with a screenshot, or a reproducible bug with a proposed expected result.

Avoid starting with a broad request such as “modernise the web UI” or “add support for a new board” unless a maintainer has first split it into a manageable task.

## You are welcome to contribute in other ways

- Test a branch or pull request and report exactly what happened.
- Improve documentation, translations, screenshots, or reproducible examples.
- Help a maintainer clarify an existing issue.

Please never put passwords, access tokens, Wi-Fi credentials, or private logs into an issue or pull request.
